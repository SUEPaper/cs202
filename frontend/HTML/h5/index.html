HTML5 新特性

新语义元素​

HTML5引入了一些新的语言义元素，用于更准确地描述文件结构和内容。这些新的语言义元素有助于开发人员和搜索引擎更好地解析和解析网页。

header、nav 和 footer​

HTML5中的<header>、<nav>并且<footer>是用于确定网页结构的语言义元素。 1、<header>元素表示文档或节的引导部分，通常包含网站的标题、标签、引导​​栏等内容。它可以出现在文档的顶部或节的开头处。

<header>
  <h1>网站标题</h1>
  <nav>
    <ul>
      <li><a href="#">首页</a></li>
      <li><a href="#">关于我们</a></li>
      <li><a href="#">联系我们</a></li>
    </ul>
  </nav>
</header>

2、<nav>元素用于表示导航链接的部分。它应包含网站的主要导航菜单或链接集合。

<nav>
  <ul>
    <li><a href="#">首页</a></li>
    <li><a href="#">关于我们</a></li>
    <li><a href="#">联系我们</a></li>
  </ul>
</nav>

3、<footer>元素表示文件或节的页脚部分，通常包含版本权限信息、联系方式、相关链接等内容。它可以出现在文件的底部或节的连接处。

<footer>
  <p>&copy; 2023 公司名称. 版权所有.</p>
  <p>联系我们: contact@example.com</p>
</footer>

这些语言义元素有助于提供高网页的可读性和可访问性，而且在搜索引擎优化方面也有一定的影响，因为它们可以提供更准确信息给搜索引擎解析网页结构和内容。

article、section 和 aside​

HTML5中的<article>、<section>并且<aside>是用于组织和标记内容的语言义元素。 1、<article>元素用于表示一个独立的、完整的、可以单独分配的内容元。它通常表示一篇博客文章、新闻报道、论坛帖子等独立立的内容块。

<article>
  <h2>文章标题</h2>
  <p>文章内容...</p>
</article>

2、<section>元素表示文档中的一个独立节或块。它可以用于将相关的内容分组，通常具有一个标题。元素不应该单独使用，而是应该根据内容的<section>综合结构来使用的。

<section>
  <h2>节标题</h2>
  <p>节内容...</p>
</section>

3、<aside>元素表示与页面内容相关但可以被视为独立的部分。它通常用于包含侧面栏、广告、附件、引用等内容。元素的内容可以是任选的，如果被移走，页面的主要<aside>内容仍然可以单独存在。

<aside>
  <h3>侧边栏</h3>
  <ul>
    <li>链接1</li>
    <li>链接2</li>
    <li>链接3</li>
  </ul>
</aside>

这些语言义元要素有助于更好地组织和描述网页的内容结构。它们提供了更明确的语言义信息，有助于搜索引擎和辅助技术更好地理解和解析网页。此外，使用这些元素还能增加可访问性和提升网页的可读性。

time、mark 和 progress​

HTML5中的<time>、<mark>并且<progress>是用于标记特定类型内容的语言义元素。 1、<time>元素用于表示日期和时间。它可以包含具体的日期、时间或日期时间的组合。这个元素对机器和辅助技术来说非常有用，因为它们可以更容易地解析和解析时间信息。

<p>会议时间：<time datetime="2023-06-10T10:00">2023年6月10日 上午10:00</time></p>

2、<mark>元素用于突显文本中的关键字或高亮显示的内容。它通常用于引读者的注意或在搜索结果中突显相关关内容。

<p>这篇文章中的关键信息：<mark>重要内容</mark></p>

3、<progress>元素表示任务的完成度。它通常用于显示操作、上传、下载等任务的进度。通过设置value和max属性，可以指定当前的进行度和最大进度。

<progress value="50" max="100"></progress>

这些语言义元素有助于提供更准确的信息和下一篇文章，并使网页内容更易于理解和解析。它们有助于改进可访问性、搜索引擎优化和用户体验。

多媒体标签：audio、video​

HTML5提供了<audio>和<video>标签，用于在网页中嵌入音频和视频内容。 1、<audio>元素用于填充音频内容。可以通过指定src属性来引用音频文件的URL，并使用controls属性显示音频控制条。

<audio src="audio-file.mp3" controls></audio>

可以在<audio>标签内添加多个<source>元素，以支持不同格式的音频文件，浏览器将根据支持的格式自动选择合适的文件。

<audio controls>
  <source src="audio-file.mp3" type="audio/mpeg">
  <source src="audio-file.ogg" type="audio/ogg">
  Your browser does not support the audio tag.
</audio>

2、<video>元素用于填充视频内容。可以通过指定src属性来引用视频文件的URL，并使用controls属性显示视频控制条。

<video src="video-file.mp4" controls></video>

与<audio>类似的，可以在<video>标签内使用多个<source>元素来支持不同格式的视频文件。

<video controls>
  <source src="video-file.mp4" type="video/mp4">
  <source src="video-file.webm" type="video/webm">
  Your browser does not support the video tag.
</video>

除掉src和controls属性，还可以利用其他属性来制作音频和视频的播放，如autoplay（自动播放）、loop（循环播放）、poster（封面图像）等。

这些标签提供了一种方便的方式来在网页中嵌入音频和视频内容，用户可以直接在浏览器中播放和控制多媒体。

HTML5 Web 存储​

localStorage 和 sessionStorage​

HTML5中的localStorage和sessionStorage是Web Storage API的一部分，用于在客户端（即浏览器）中存储数据。它们都是基于按键值的存储机制，可以用于在浏览器会话之间保持久保存数据。

localStorage 和 sessionStorage 都是 HTML5 Web 存储 API 所提供的客户端存储方式，它们的主要区别在于数据的生命周期和作用域。

localStorage 通过调用 localStorage 对象来存储数据，可以将数据永久保存在浏览器中，即使关闭浏览器后再次打开也不会丢失。localStorage 适合存储长期需要保留的数据，如用户配置、偏好设置等。

sessionStorage 同样使用 sessionStorage 对象存储数据，但与 localStorage 不同，数据只存在会话期间，即只存在于当前浏览器窗口或选项卡中。当用户关闭浏览器窗口或选项卡后，数据将被删除。sessionStorage 适合存储临时性的数据，如正在进行的表单数据、暂存的文本、操作步骤等。

无论是 localStorage 还是 sessionStorage，数据都是以键值对的形式存储的，可以通过 JavaScript 的 setItem、getItem、removeItem 方法进行操作。同时它们的数据类型都限制为字符串，如需存储其他数据类型需要进行序列化和反序列化。
